こんな時に役立つ： Claude Code を仕事で使っていて、「先週の更新で何が壊れた？」「この hook は本当に効いているのか？」「トークン消費が突然倍になったが、何が原因か？」と毎月のように振り返る人。GitHub Issue を全部読む時間はないが、起きている事故と対処は知っておきたい人。

何を届けるか（毎月）

  • 1. 月次インシデント・ラウンドアップ
    その月に Claude Code 公式リポジトリで報告された事故・退行・仕様変更を 4-8 件、再現手順と対処方法と一緒に整理する。Issue 番号、発生のきっかけ、影響範囲、回避策をまとめる。
  • 2. 安全チェックリスト
    料金、秘密情報、破壊的操作、長時間セッション、公開前確認の 5 領域の月次更新版。前月との差分が分かるように書く。
  • 3. コピペで使える hook 例
    毎月 1-2 個、その月の事故から逆算した PreToolUse / Stop hook を出す。動作確認済みの bash スクリプトと、適用方法を添える。
  • 4. 失敗事例 1 件の深掘り
    月で最も影響の大きかった事故 1 件を、症状・原因・復旧手順・予防策・月次チェック項目の追加 まで分解する。GitHub Issue で「同じ症状」と言われている関連報告も束ねる。
  • 5. 既存商品の更新案内
    Token Book、Migration Playbook、Postmortems 本、Survival Guide、cc-safe-setup の各商品で、その月にどんな更新が入ったかを整理する。読み返すべき章があればその案内も書く。

これは何ではないか（誠実な開示）

含まれないもの：

  • 1 対 1 のサポートやチャット相談（コンテンツ商品としての提供）
  • 専用 Discord や Slack（個別の議論場は用意していない）
  • Anthropic 公式との特別な連携や事前情報（一次情報は誰でも見られる Issue を整理する立場）
  • 「絶対に事故が起きなくなる」保証（事故を完全に防ぐ仕組みではなく、起きたものを早く知り、対処を共有するもの）

合うかどうかの目安： Claude Code を週 5 日以上業務で使っており、毎月の事故と仕様変更を 30 分以内で把握したい人には合う。月 1 回しか触らない人や、事故が起きても影響が小さい使い方の人には合わない。

よくある質問

Q. Token Book を持っているが、Safety Lab とどう違うか？
A. Token Book は一度買えば全章読める一括商品。Safety Lab は毎月新しい事故と対処を追跡する月次の購読商品。Token Book を読んだあと、最新の事故事例と仕様変更を継続して受け取りたい人に Safety Lab が合う。
Q. 解約はいつでもできるか？
A. Ko-fi 側でいつでも解約可能。次の月から請求が止まる。日割り返金はない（Ko-fi の仕様）。
Q. 過去号は読めるか？
A. 参加した月以降の号にアクセスできる。過去号のアーカイブは現時点では未提供。月額メンバーが増えてきた段階でアーカイブ提供を検討する。
Q. 何で配信されるか？
A. Ko-fi のメンバー限定投稿として、月 1 回 PDF か Markdown でまとめて公開する。ファイルはダウンロードして手元に残せる。
Q. 月の途中で参加した場合、その月号は読めるか？
A. 参加した日以降に投稿される号から読める。参加月の号がまだ未投稿なら、それも含む。投稿済みの当月号には間に合わない場合がある。
